## Root Directory


### Configuration & Build Files

| File / Folder | Purpose |
|---|---|
| `app.json` | Expo app manifest вҖ” app name, slug, version, icon, splash screen, plugins, platform-specific configs (iOS/Android/Web) |
| `eas.json` | EAS Build profiles вҖ” defines `development`, `preview`, and `production` build configurations for APK/AAB generation |
| `drizzle.config.ts` | Drizzle Kit configuration вҖ” points to schema files, sets the database output directory for migrations |
| `tailwind.config.js` | NativeWind / Tailwind CSS configuration вҖ” brand colors, fonts, and theme extensions |
| `babel.config.js` | Babel configuration for Expo (preset, plugins) |
| `metro.config.js` | Metro bundler configuration вҖ” module resolution and asset handling |
| `postcss.config.mjs` | PostCSS config for Tailwind CSS processing |
| `tsconfig.json` | TypeScript compiler options вҖ” strict mode, path aliases, target settings |
| `eslint.config.js` | ESLint rules for code quality enforcement |
| `expo-env.d.ts` | TypeScript ambient type declarations for Expo environment |
| `nativewind-env.d.ts` | TypeScript ambient type declarations for NativeWind Tailwind classes |
| `global.css` | Global CSS file imported for Tailwind utility classes |

### Package Management

| File | Purpose |
|---|---|
| `package.json` | Project manifest вҖ” dependencies, scripts, version. Managed with pnpm |
| `pnpm-lock.yaml` | pnpm lockfile for deterministic dependency resolution |
| `package-lock.json` | npm lockfile (legacy, project uses pnpm) |
| `pnpm-workspace.yaml` | pnpm workspace config |

### Documentation

| File | Purpose |
|---|---|
| `README.md` | Project overview, feature list, tech stack, setup instructions, environment variables |
| `PROJECT_WALKTHROUGH.md` | Detailed walkthrough вҖ” problem statement, architecture, database design, user workflows, admin system |
| `PROJECT PROPOSAL.pdf` | Original project proposal document |

### `src/db/` вҖ” Database Layer

Drizzle ORM schema, migrations, relations, and seed data.

```
src/db/
в”ңв”Җв”Җ index.ts               # Database connection вҖ” Neon PostgreSQL client setup
в”ңв”Җв”Җ schema/
в”Ӯ   в”ңв”Җв”Җ index.ts           # Schema barrel export вҖ” re-exports all table definitions
в”Ӯ   в”ңв”Җв”Җ auth.ts            # Auth-related tables вҖ” user, session, account, verification (Better Auth schema)
в”Ӯ   в””в”Җв”Җ items.ts           # Application tables вҖ” items, claims, item_images, notifications, admin_logs, saved_items
в”ңв”Җв”Җ relations.ts           # Drizzle relational definitions вҖ” defines table relationships (1:N, N:M)
в”ңв”Җв”Җ seed.ts                # Database seeder вҖ” inserts sample data for development
в””в”Җв”Җ clear.ts               # Database clearer вҖ” truncates all tables (dev utility)
```

**Key tables defined in `schema/items.ts`:**
- `items` вҖ” Lost/found reports with status workflow (PENDING вҶ’ PUBLISHED вҶ’ CLAIMED/RESOLVED)
- `claims` вҖ” Ownership claims on items with proof and pickup verification
- `itemImages` вҖ” Associated images for each item
- `notifications` вҖ” In-app notification records
- `adminLogs` вҖ” Audit trail for all admin mutations
- `savedItems` вҖ” User bookmarked/saved items
